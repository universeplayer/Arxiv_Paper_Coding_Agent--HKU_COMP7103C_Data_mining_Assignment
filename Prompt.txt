最终产物 **必须且只能包含** 以下 4 个文件，并全部放在同一目录中：
- `README.md`

直接用浏览器打开 `index.html` 即可运行。

## 🛑 反复制 / 反糊弄规则（强制）

**文件语域必须严格区分：**

- `README.md`

- `index.html`
  - 只写 HTML

- `styles.css`
  - 只写 CSS
  - 覆盖页面全部样式

- `main.js`
  - 只写 JavaScript

  - 必须注册 `DOMContentLoaded` 入口并实际执行初始化流程

## 🎯 功能目标（必须实现）

### 1. 论文列表（首页）

- 展示 arXiv Computer Science 最新论文
- 固定分类：
  - `cs.AI`, `cs.CL`, `cs.CV`, `cs.LG`
- 顶部分类筛选：
  - `All`, `cs.AI`, `cs.CL`, `cs.CV`, `cs.LG`
  - 点击后立即过滤列表
  - 激活态明显
  - 从详情页返回时保持上一次筛选状态

### 2. 详情页（SPA + hash 路由）

- 使用 `window.location.hash`
- 路由规则：
  - 列表页：空 hash / `#home` / `#cat/<code>`
  - 详情页：`#paper/<arxivId>`
- 详情页必须显示：
  - 标题
  - 作者列表
  - 完整摘要
  - 主分类
  - 提交时间
  - **Open PDF 按钮**

**Open PDF 规则（必须）：**
- 始终使用论文对象中的 `pdfUrl`
- 新标签页打开
- 不允许拼接当前页面 URL 或 hash

## 🌐 数据获取与解析（必须稳定）

### 数据源
- 使用 arXiv 官方 Atom API（CS 分类，50 篇，按 submittedDate 倒序）

### 请求顺序（严格）

按顺序尝试以下 3 个 endpoint（任一成功即停止，每一步必须有超时控制）：
1. `https://cors.isomorphic-git.org/<原始 arXiv URL>`
2. `https://api.allorigins.win/raw?url=<URL 编码后的原始 URL>`
3. 直接请求 arXiv 官方 API（HTTPS）

### 失败处理
- 若全部失败：
  - 页面显示错误状态
  - 保持 UI 结构
  - 不渲染示例论文
  - 在控制台输出 `console.warn`

### Atom 解析
使用浏览器 XML 解析器，对每个 `<entry>` 提取：
- `arxivId`
- `title`
- `summary`
- `authors`（数组）
- `primaryCategory`
- `published`
- `pdfUrl`（优先 API 提供，否则标准 arXiv PDF URL）

单条解析失败不得导致整体崩溃。
